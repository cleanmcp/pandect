# pandect

> Opinionated rulebooks for AI coding agents. One `.md` per stack. Hand it to Claude Code, Codex, or Cursor with your idea — they bootstrap, build, and ship.

## Why

Non-coders and senior devs alike waste hours on the same questions every new project: what package manager, which test runner, how do I deploy, what does my AI agent need to know? `pandect` packages that decision-tree into one self-contained markdown file per stack — every version cited, every config inlined, every command copy-pasteable. The agent reads it once and stops asking.

The name comes from the legal term *pandect*: a comprehensive single-volume treatise that covers an entire body of law. That's what each rulebook is — a complete code for one stack.

## Quick start

```sh
# pick a stack — drops RULEBOOK.md into your current folder
npx pandect nextjs

# then start a Claude Code / Codex / Cursor session in that folder.
# tell the agent to read RULEBOOK.md, hand it your product idea, watch it ship.
```

Or, no Node required — direct download via curl:

```sh
curl -fsSL https://raw.githubusercontent.com/cleanmcp/pandect/main/rulebooks/nextjs.md -o RULEBOOK.md
```

## How a rulebook is structured

Every rulebook follows the same 21-section spec:

1. Snapshot — versions table, decisions table
2. Zero-to-running — setup per OS
3. Project Layout — naming + folder map
4. Architecture — process boundaries + data flow
5. Dev Workflow
6. Testing & Parallelization
7. Logging
8. AI Rules — ALWAYS / NEVER / Blast Radius / Definition of Done / Self-Verification / Parallelization
9. Stack-Specific Pitfalls
10. Performance Budgets
11. Security
12. Deploy / Publish
13. Claude Code Integration — full `CLAUDE.md` + `.claude/settings.json`
14. Codex Integration — full `AGENTS.md` + `.codex/config.toml`
15. Cursor / VS Code — `.cursor/rules`, `extensions.json`, `launch.json`
16. First-PR Scaffold — every file to create, in order
17. Idea → MVP Path
18. Feature Recipes
19. Troubleshooting
20. Glossary
21. Update Cadence

If it's not in the rulebook, the agent will guess. So nothing is missing.

## CLI usage

```sh
npx pandect <stack>              # writes ./RULEBOOK.md
npx pandect <stack> -o foo.md    # writes ./foo.md
npx pandect --list               # one stack per line
npx pandect --version
npx pandect --help
```

Or install globally:

```sh
npm i -g pandect
pandect nextjs
```

## Contributing

Each rulebook is generated by feeding `RULEBOOK_GENERATOR.md` (the meta-prompt) into a fresh AI session per stack. To propose a new stack, open an issue with the stack description and target OSes; to update an existing one, regenerate via the meta-prompt and PR the diff.
